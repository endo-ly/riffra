# Riffra 製品構想

## 1. 製品の定義

Riffraは、演奏、録音、音声素材、数式による信号生成から自分の音を作り、その音を再利用可能な素材として保存し、音楽の中で試せるWindows向け制作ワークベンチである。

中心となる価値は、次の三つを一つの制作環境でつなぐことにある。

1. 楽器や声を入力し、リアルタイムに音を作る
2. 素材の加工と信号の生成によって、新しい音源を設計する
3. 作った音を組み合わせ、音楽として成立するか確かめる

ギターとVST3は、製品価値を最も分かりやすく体験できる入口である。ただし、扱う入力をギターに限定しない。歌声、ベース、マイク収録した楽器、外部シンセサイザー、MIDI演奏、既存の音声ファイル、数式から作る音も同じ制作循環へ取り込む。

## 2. 解決する問題

音楽制作では、演奏、音作り、録音、波形加工、音源設計、素材管理、簡単な編曲が別々の道具へ分かれやすい。その結果、次の問題が起こる。

- 良い音ができても、設定と録音の関係が残らない
- 元の演奏と加工後の音を安全に行き来できない
- 作った音を曲の中で試すまでに、多くの準備が必要になる
- 録音や加工結果が散らばり、再利用できなくなる
- 数式や変調から作った音が、試作のまま演奏や楽曲へつながらない
- 高機能なDAWを開くほどではない着想を、すぐ形にできない

Riffraは、音を作った時点で終わらせず、由来と設定を保った素材として蓄積し、演奏と編集の間を往復できるようにする。

## 3. 三つの制作領域

Riffraの制作画面は、Play、Design、Arrangeの三領域で構成する。

| 領域    | 目的                       | 主な入力                             | 主な成果                              |
| ------- | -------------------------- | ------------------------------------ | ------------------------------------- |
| Play    | 演奏・歌唱しながら音を作る | 楽器、声、外部音源、MIDI             | 録音、音色、ラック、スナップショット  |
| Design  | 音の材料と構造を設計する   | 音声素材、数式、波形、ノイズ、分離音 | ワンショット、ループ、音源、派生素材  |
| Arrange | 作った音を組み合わせる     | 録音、音源、MIDI、分離音             | スケッチ、ミックス、WAV、ステム、MIDI |

三領域は工程順に固定しない。どこから始めてもよく、制作中に自由に往復できる。

```text
                 Play
        演奏しながら音を作る
             ↙         ↘
            ↙           ↘
       Design  ←──────→  Arrange
    音を設計する         音を組み合わせる
```

### 3.1 Play

Playは、入力された音を聞きながら、その場で演奏に使う音を決める場所である。

主な用途は次のとおり。

- ギターやベースをアンプシミュレーターへ通す
- 歌声へコンプレッサー、イコライザー、空間系処理をかける
- マイク収録した楽器を監視しながら音を整える
- 外部シンセサイザーやドラムマシンを加工する
- MIDI鍵盤からVST3音源や内蔵音源を演奏する
- Designで作った音源を鍵盤やパッドから演奏する

Playが担う機能は次のとおり。

- 音声とMIDIの入力・出力選択
- 低遅延の監視
- VST3の検索、読込み、画面表示、音声処理
- プラグインと内部処理を並べるラック
- バイパス、ミュート、音量、パン
- 音色の保存、比較、再利用
- 原音と加工音の同時録音
- 入出力メーターと安全保護

Playでは、波形の細かな編集よりも、演奏への反応、音の変化、操作の速さを優先する。

### 3.2 Design

Designは、音の材料と構造を設計し、再利用できる音源へ仕上げる場所である。出発点は、既存の素材に限らない。録音や音声ファイルを加工する方法と、数式や基本信号から音を生成する方法を、同じ重要度で扱う。

| 作り方   | 出発点                         | 主な操作                           | 成果                       |
| -------- | ------------------------------ | ---------------------------------- | -------------------------- |
| 素材加工 | 録音、音声ファイル、分離音     | 切出し、変形、重ね合わせ、割当     | ワンショット、ループ、音源 |
| 信号生成 | 数式、基本波形、ノイズ、制御値 | 合成、変調、包絡、フィルター、接続 | 音色、演奏可能な音源、音声 |

二つの作り方は途中で合流できる。生成した音を音声として加工したり、音声素材を生成器の変調源として使ったりしてよい。

数式による生成は、任意のプログラムを実行する機能ではない。音声信号に必要な演算と接続だけを安全な範囲で提供し、計算量と出力を制御できる音源設計として扱う。

#### 素材を加工して作る

- 録音した一音を切り出してワンショットにする
- 開始位置、終了位置、ループ、フェード、包絡を非破壊で整える
- ピッチ、長さ、音量、定位を調整し、複数の素材を重ねる
- 音源を分離し、必要な成分を新しい素材にする
- 音声を解析し、加工前後や参照音源との差を確認する

#### 信号を生成して作る

- 時間、音程、強さなどを変数とする数式から波形を作る
- 基本波形やノイズを組み合わせ、別の信号で変調する
- 生成器、包絡、フィルター、音声処理の接続を組み立てる
- 作った構造を鍵盤やMIDIから演奏し、その場で調整する

#### 音源として仕上げる

- パッド、鍵盤、音域、強さへ音を割り当てる
- VST3やラックを通した結果を新しい素材として録音する
- ワンショット、ループ、キット、音源、音色として保存する
- 元素材または生成定義と、処理内容、生成物の関係を残す
- 生成式、接続、パラメーター、乱数条件を含む音色の保存

波形編集、音源化、分析、参照比較、音源分離、信号生成は、独立した制作領域ではなく、Designで音を設計するための道具として扱う。

### 3.3 Arrange

Arrangeは、PlayとDesignで作った音を時間軸へ置き、音楽の中で使えるかを確かめる場所である。

主な用途は次のとおり。

- 録音した演奏を並べる
- 作ったワンショットやループを組み合わせる
- MIDIで音源を演奏させる
- 音量、定位、フェード、タイミングを調整する
- 複数の音が同時に鳴ったときの関係を確認する
- 完成した範囲をWAV、ステム、MIDIとして書き出す

Arrangeが担う機能は次のとおり。

- 音声クリップとMIDIクリップ
- トラック、グループ、リターン、マスター
- 移動、複製、分割、範囲変更、ループ
- フェード、音量、パン、ミュート、ソロ
- テンポ、拍子、マーカー、ループ範囲
- 基本的な自動変化
- マスター、トラック、ステムの書出し
- 外部DAWへ渡すための音声、MIDI、制作情報の出力

ArrangeはRiffraで作った音を音楽的な文脈で確かめるための小規模な制作環境である。大規模な商用DAWの全機能を置き換えることは目的としない。

## 4. 制作の循環

### 4.1 演奏から始める

```text
楽器・声
  ↓
PlayでVST3やラックを使って音を作る
  ↓
原音と加工音を録音する
  ↓
Designで切出し・加工・音源化する
  ↓
Arrangeで曲の中に置く
```

### 4.2 数式から始める

```text
数式・基本波形・ノイズ
  ↓
Designで合成・変調・包絡・フィルターを組み立てる
  ↓
Playで演奏しながら音色を調整する
  ↓
音源または音声素材として保存する
  ↓
Arrangeで組み合わせる
```

### 4.3 素材から始める

```text
録音・音声ファイル・分離音
  ↓
Designで加工・分離・割当を行う
  ↓
Playで楽器として演奏する
  ↓
Arrangeで組み合わせる
```

### 4.4 曲から音へ戻る

```text
Arrangeで音の問題を見つける
  ↓
Playで弾き直す、またはDesignで加工し直す
  ↓
新しい素材として保存する
  ↓
Arrangeへ差し替える
```

## 5. 共有する制作基盤

三領域は別々のファイルや設定を持たず、同じセッション、素材、履歴を共有する。

| 概念            | 意味                                                                                     |
| --------------- | ---------------------------------------------------------------------------------------- |
| Scratch Session | 起動直後から使える作業空間。プロジェクト作成前の着想も自動保存する                       |
| Project         | セッション、素材参照、時間軸、音色、設定、履歴をまとめた保存単位                         |
| Asset           | 録音、音声、生成定義、音源、音色、ラック、MIDI、分離音、書出し結果などの再利用可能な素材 |
| Library         | 素材を種類にかかわらず検索し、由来と関連を辿る索引                                       |
| Inbox           | 未整理の録音、取込み、分離結果などを失わず受け止める領域                                 |
| Rack            | 入力から出力までの音声処理を並べた経路                                                   |
| Recording       | 原音、加工音、MIDI、録音条件をまとめた記録                                               |
| Take Group      | 同じ意図で録音した複数候補のまとまり                                                     |
| Clip            | 素材を変更せず、使用範囲と配置を示す時間軸上の参照                                       |
| Snapshot        | ラックや音色の状態を保存し、比較できる候補                                               |
| Job             | 解析、分離、走査、書出しなど、音声入出力を止めずに行う処理                               |

### 5.1 素材の由来

素材は原本を上書きせず、元の素材との関係を保持する。

```text
録音した原音
  ├─ 加工した録音
  ├─ 切り出したワンショット
  ├─ 作成したループ
  ├─ 分離した音
  ├─ 鍵盤・パッド用の音源
  └─ 曲へ配置したクリップ
```

数式から作った音も、生成式だけを保存して終わりにしない。音の構造、パラメーター、乱数条件、使用した処理、生成した音声、演奏先の関係を辿れるようにする。

各素材には、元の素材、使用したラック、音色、処理内容、作成日時、使用先を辿れる情報を残す。

### 5.2 履歴と再利用

- 操作は可能な範囲で取り消し、やり直しできる
- 音色や素材の候補を比較しても、比較前の状態を失わない
- 良い結果は名前を決める前でもInboxへ残す
- ファイルが移動または消失しても、残りの制作内容を開ける
- プラグインがない環境では、無効な置き場所または書出し音声で制作意図を残す

## 6. 画面構成

### 6.1 最上位の移動

最上位の制作領域はPlay、Design、Arrangeの三つとする。

Homeは制作領域と並列に扱わず、起動、最近の作業、機器状態、復旧、設定への入口とする。

### 6.2 共通領域

| 画面部分  | 役割                                                   |
| --------- | ------------------------------------------------------ |
| 上部バー  | セッション名、保存状態、音声状態、安全操作、検索、設定 |
| Library   | 素材とプラグインを探し、各制作領域へ投入する           |
| Inspector | 選択した対象の詳細と編集項目を表示する                 |
| Transport | 再生、停止、録音、位置、テンポ、ループを操作する       |

選択、ドラッグ、試聴、取り消し、エラー表示の意味は、制作領域が変わっても統一する。

### 6.3 補助機能の位置

- 波形編集、信号生成、音源化、分析、参照比較、音源分離はDesignに置く
- 録音操作はPlayを中心に、Arrangeからも同じ状態で利用できる
- AI支援は独立した制作領域にせず、選択中のPlay、Design、Arrangeを補助する
- LibraryとInspectorは三領域で共有する
- 未完成の機能を、利用可能な制作領域と同じ強さで表示しない

## 7. 製品原則

### 7.1 音が出ることを最優先する

画面や設定項目の存在ではなく、入力した音が意図した経路を通り、安全に聞こえることを優先する。

### 7.2 準備より先に演奏できる

起動時にScratch Sessionを開き、プロジェクト作成を求めずに演奏と録音を始められるようにする。

### 7.3 原本を壊さない

録音や取込み素材を直接上書きせず、編集内容は参照、設定、または新しい素材として保存する。

### 7.4 良い結果を素材として残す

録音、音色、切出し、分離結果、分析、書出しを、後から検索して再利用できる状態にする。

### 7.5 値だけでなく意図を残す

数値だけでなく、使用した機器、ラック、音色、元素材、変更理由、使用先を保存する。

### 7.6 失敗を隠さない

失敗を成功として扱わず、影響範囲、守られたデータ、利用できる復旧操作を示す。復旧不能な処理を無理に継続しない。

### 7.7 中核機能は手元で動く

演奏、録音、加工、配置、保存は外部サービスなしで利用できる。外部送信は対象と目的を示し、明示的な許可を必要とする。

### 7.8 複雑さを段階的に見せる

基本操作では安全な既定値を使い、必要な利用者だけが詳細な経路、パラメーター、保存情報へ進めるようにする。

## 8. 製品の範囲

### 8.1 中核範囲

- Windows上の低遅延な音声入出力
- VST3の走査、読込み、画面表示、音声処理、状態保存
- 演奏中のラック編集と音色比較
- 原音と加工音の録音
- 非破壊の波形加工と音源化
- 数式、基本波形、ノイズからの信号生成と音源化
- 生成器、変調、包絡、フィルターを組み合わせる音源設計
- 生成定義の保存、再現、演奏、音声素材への変換
- 音声とMIDIの基本的な配置
- 短い楽曲やスケッチの作成
- マスター、ステム、MIDIの書出し
- セッション、素材、履歴、復旧情報の保存

### 8.2 外部DAWへ任せる範囲

- 大規模なトラック構成と複雑な経路設定
- 楽譜制作と映像同期
- 高度なコンピングと編集管理
- 大規模な自動変化の編集
- 専門的なミックスとマスタリング
- 多人数の共同編集

Riffraは外部DAWとの競合だけを目的とせず、作った音をWAV、ステム、MIDI、制作情報として安全に渡せるようにする。

### 8.3 拡張範囲

- ウェーブテーブル、粒状合成、物理モデルなどの高度な合成方式
- 複数素材を使う本格的な音源作成
- 高度な参照比較と音色探索
- 制作意図を理解するAI支援

拡張機能もPlay、Design、Arrangeと共通の素材・履歴モデルを使い、独立した目的のない画面を増やさない。

## 9. 代表的な完成像

### 9.1 ギター

1. オーディオインターフェースを選ぶ
2. ギター入力と出力を選ぶ
3. AmpliTubeなどのVST3を読み込む
4. プラグイン画面を開き、弾きながら音を調整する
5. 原音と加工音を録音する
6. 録音を聞き比べ、必要ならDesignで加工する
7. Arrangeへ置き、曲の中で確認する
8. セッションを開き直して同じ音へ戻る

### 9.2 歌声

1. マイク入力を選ぶ
2. 声を聞きながら処理の順序と量を調整する
3. 原音と加工音を録音する
4. 良い部分をDesignで切り出す
5. Arrangeで伴奏と組み合わせる

### 9.3 数式から作る音源

1. Designで数式、基本波形、ノイズから音の出発点を作る
2. 変調、包絡、フィルターを組み合わせる
3. Playで鍵盤やMIDIから演奏し、反応を調整する
4. 生成式、接続、パラメーターを音色として保存する
5. 必要なら音声素材として書き出し、Arrangeでほかの音と組み合わせる

### 9.4 素材から作る音源

1. 録音または音声ファイルをDesignへ取り込む
2. 波形を切り、整え、必要な加工を行う
3. ループ、パッド、鍵盤用音源として保存する
4. Playで演奏して反応を確かめる
5. Arrangeでほかの音と組み合わせる

三つの完成像が同じセッション、素材、履歴の上で成立することが、Riffraの製品価値である。
