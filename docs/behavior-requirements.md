# Riffra 製品挙動要件

## 1. 要件の表し方

製品の挙動は、実装方法ではなく、利用者が行う操作と確認できる結果で定義する。

各要件は次の要素を持つ。

- **条件**: 操作を始める前の状態
- **操作**: 利用者またはシステムが行うこと
- **期待結果**: 操作後に成立しなければならない事実
- **確認対象**: 音、画面、保存データ、履歴、外部機器のどこで確認するか

優先度は次の意味を持つ。

| 優先度 | 意味                                                 |
| ------ | ---------------------------------------------------- |
| P0     | 中核の制作体験、安全性、データ保全に不可欠           |
| P1     | 日常的な制作に必要だが、限定した代替手段を許容できる |
| P2     | 高度な制作や効率化を支える拡張機能                   |

要求IDは、設計、実装、テスト、適合判定で共通して使用する。現在の判定と証拠は[挙動確認・課題管理表](./behavior-verification.md)に記録する。

## 2. 三領域で共有する基本挙動

- **G-001 [P0] Scratch Sessionを即時に開く** — 初回起動またはプロジェクト未指定で起動すると、作成画面を挟まず編集可能なScratch Sessionが開く。確認対象は初期画面と保存されたセッションである。
- **G-002 [P0] 制作状態を共有する** — Play、Design、Arrangeを切り替えても、セッション、選択、再生位置、録音、ミュート、素材参照が失われない。
- **G-007 [P1] 集中表示は制作状態を変えない** — 集中表示を切り替えると補助領域だけが隠れ、音声、セッション、履歴は変化しない。

## 3. Play：演奏・歌唱しながら音を作る

### 3.1 中核フロー

- **FLOW-001 [P0] 起動して演奏する** — 保存済みの機器設定とラックがある状態で起動すると、それらが復元され、安全な音量から演奏を始められる。
- **FLOW-002 [P0] 音色候補を比較する** — ラックに複数の候補がある状態で切り替えると、編集中の状態を失わず、音量差を抑えて比較できる。
- **FLOW-003 [P0] 思いつきを録る** — 音声機器が利用可能な状態で録音を開始・停止すると、原音、加工音、録音条件がInboxへ保存される。

### 3.2 音声入出力

- **AUD-001 [P0] 音声ドライバーと入出力を列挙する** — Windowsで利用可能な音声方式、機器、入力数、出力数を表示する。
- **AUD-002 [P0] サンプルレートとバッファーを適用する** — 値を変更すると一時的にミュートし、実際に採用された値を画面とセッションへ反映する。
- **AUD-003 [P0] 入出力をメーター表示する** — 機器が動作している間、入力と出力のピーク、クリップ、無効値、遅延を更新する。
- **AUD-005 [P1] 処理遅延を補償する** — 遅延が異なる処理を組み合わせても、演奏、録音、時間軸、書出しの位置関係を維持し、補償できない場合は明示する。

### 3.3 MIDI演奏

- **MIDI-001 [P0] MIDI機器を検出して開閉する** — 機器の接続と切断に応じて一覧を更新し、開閉結果を表示する。
- **MIDI-002 [P1] MIDI演奏を処理・記録する** — ノート、強さ、サステイン、ピッチなどを音源へ渡し、録音時には再現可能なMIDIデータを保存する。
- **MIDI-003 [P0] Panicで発音を停止する** — Panic操作またはMIDI機器の異常時に、鳴り続けているすべての音を停止する。

### 3.4 VST3とラック

- **PLG-001 [P0] VST3の走査を隔離する** — 正常なプラグインと異常なプラグインが混在しても、画面と音声を停止させず、問題の対象と理由を特定する。
- **PLG-002 [P1] プラグインを検索する** — 名前、提供元、分類、お気に入り、安定性で絞り込むと、該当する結果だけを表示する。
- **PLG-003 [P0] プラグインを読み込み、迂回し、取り外す** — ラック表示、実際の音声処理、迂回状態、取外し結果が一致する。
- **PLG-004 [P1] プラグインのパラメーターを操作する** — 画面で値を変更すると、プラグインの実値、表示値、保存値が一致する。
- **PLG-005 [P0] プラグイン状態を保存・復元する** — パラメーター、内部プリセット、迂回、内部状態を保存し、再起動後に復元する。復元不能なら無効な置き場所として残す。
- **RACK-001 [P1] 音の経路を表示する** — 入力から出力までの順序、分岐、迂回、ミュート、チャンネル数、異常箇所を読み取れる。
- **RACK-002 [P1] ラックを編集・再利用する** — 追加、削除、複製、並替え、部分保存を行うと、実際の接続と保存状態が一致する。
- **RACK-003 [P1] 一つの操作を複数パラメーターへ割り当てる** — 範囲、反転、変化曲線を設定し、安全な値の範囲で複数項目を同時に操作できる。
- **RACK-004 [P1] 音色候補を保存・比較する** — 保存、名称変更、タグ、比較、試聴を行っても、比較前の状態を失わない。
- **RACK-005 [P2] 音色の変化案を安全に作る** — 固定項目と変更可能項目を分け、危険な値を避けながら候補を作り、採用、破棄、取消しができる。
- **RACK-006 [P1] プラグインなしでも聞ける代替音声を作る** — ラックやMIDIを保持したまま加工音を書き出し、プラグインがない環境でも当時の音を確認できる。

### 3.5 録音

- **REC-001 [P0] 原音と加工音を同時録音する** — 同じ録音条件で両方のWAVと録音情報を作り、録音中は定期的に確定し、停止時に完成状態へ移す。
- **REC-002 [P1] 録音前後の範囲を制御する** — カウント、事前録音、指定範囲録音、ループを設定すると、実際の録音範囲と保存情報が一致する。
- **REC-003 [P1] 複数の録音候補を管理する** — 録音日時、入力、ラック、音色、遅延、評価、採用状態を保持し、候補を切り替えられる。

## 4. Design：音の材料と構造を設計する

### 4.1 中核フロー

- **FLOW-005 [P1] 音声から演奏可能な素材を作る** — 読み取り可能な音声を切り出し、範囲と再生方法を設定し、パッドまたは鍵盤から演奏できる素材として保存する。
- **FLOW-008 [P1] 数式から演奏可能な音源を作る** — 音声素材がない状態から信号の生成式と音の構造を組み立て、演奏して調整し、同じ音を再現できる音源として保存する。

### 4.2 信号生成

- **SYN-001 [P1] 数式から信号を生成する** — 時間、音程、強さ、利用者が定めた値を使う数式から波形を生成し、値の変更を試聴へ反映する。
- **SYN-002 [P1] 生成要素を組み合わせる** — 基本波形、ノイズ、複数の生成器、変調、包絡、フィルター、音声処理を接続し、信号の流れと各要素の値を編集する。
- **SYN-003 [P0] 生成音を安全な範囲に保つ** — 無効な式、非数、無限値、過大音量、処理時間の超過が発生しても危険な音を出さず、問題の箇所と修正方法を表示する。
- **SYN-004 [P1] 生成した音源を演奏する** — MIDIノート、強さ、離鍵、ピッチ変化を生成式と音の構造へ渡し、同時発音と発音停止を正しく処理する。
- **SYN-005 [P1] 生成定義を保存・再現する** — 生成式、接続、パラメーター、乱数条件、形式の版を保存し、開き直したときに同じ入力から同じ音を再現する。必要に応じて生成結果を新しい音声素材として保存する。

### 4.3 素材加工と音源化

- **SMP-001 [P0] 音声を非破壊で取り込む** — ワンショット、ループ、パッド、鍵盤用の設定を作っても、元の音声との参照関係を保持する。
- **SMP-002 [P1] 再生範囲、ループ、包絡を編集する** — 開始、終了、ループ、交差、ピッチ、速度、反転、包絡、フィルター、基準音を保存する。
- **SMP-003 [P1] ドラムキットを保存する** — MIDIノート、強さ、レイヤー、排他発音、順次切替、出力をまとめて再利用できる。
- **SMP-004 [P1] 鍵盤用音源を作る** — 基準音、音域、強さの範囲、レイヤー、離鍵音を設定し、MIDI演奏できる。
- **SMP-005 [P1] 複数の素材を重ねて音源を作る** — 素材ごとの音域、強さ、音量、定位、発音条件を設定し、一つの演奏可能な音源として保存する。
- **SMP-006 [P1] 基本的な音声処理を使う** — 音量、定位、イコライザー、圧縮、制限、ゲート、遅延、残響を安全な範囲で利用できる。

### 4.4 分析、参照、分離

- **ANL-001 [P0] 音声を解析する** — 波形、音量、周波数分布、位相、クリップ、強弱差、長さを、元の音声を変更せず表示する。
- **ANL-002 [P1] 参照音源との差を比較する** — 音量、長さ、周波数分布、位相などの差を、変更を伴わない情報として確認する。
- **ANL-003 [P1] 現在の音と参照音を同条件で試聴する** — 同じ開始位置と音量条件で切り替え、ループと停止を共有する。
- **SEP-001 [P0] 音源分離を後処理として実行する** — 分離中も演奏と画面操作を止めず、進捗、失敗、取消しを表示する。
- **SEP-002 [P0] 分離結果を派生素材として保存する** — 元音声、使用した処理、設定、生成ファイル、保存情報、試聴、時間軸への追加を関連付ける。

## 5. Arrange：作った音を組み合わせる

### 5.1 中核フロー

- **FLOW-004 [P0] 録音や素材を曲へ進める** — 完成した素材を時間軸へ置き、非破壊で編集し、新しい音声として書き出す。

### 5.2 時間軸と編集

- **ARR-001 [P0] 音声とMIDIを非破壊で配置する** — 移動、複製、分割、範囲変更、ループ、フェード、音量、パン、ミュートを行っても元ファイルを変更しない。
- **ARR-002 [P1] トラックとミキサーの状態を共有する** — 音量、パン、ミュート、ソロ、録音待機、ラックが時間軸とミキサーで一致する。
- **ARR-003 [P1] MIDIクリップを編集する** — 音程、開始、長さ、強さ、チャンネル、位置補正、移調、揺らぎが再生とMIDI書出しへ反映される。
- **ARR-004 [P1] テンポ、拍子、マーカー、ループ範囲を扱う** — 変更内容が音声、MIDI、格子、書出し位置へ一貫して反映される。
- **ARR-005 [P2] 時間による値の変化を編集する** — 音量、パン、ラック操作、プラグイン値を点、線、曲線で編集し、再生時に再現する。

### 5.3 書出し

- **EXP-001 [P0] マスター、トラック、ステムを書き出す** — 範囲、音量調整、形式、サンプルレート、ビット深度、チャンネル、付加情報を指定どおり反映し、元素材を変更しない。
- **EXP-002 [P1] 外部DAWへ制作情報を渡す** — トラック音声、MIDI、テンポ、拍子、マーカー、使用した音色情報を同期位置を保って出力する。

## 6. 素材、プロジェクト、制作支援

### 6.1 LibraryとInbox

- **LIB-001 [P1] 素材の種類を横断して検索する** — プラグイン、音色、ラック、音声、録音、MIDI、分離音、プロジェクト、分析、提案を同じ検索から探せる。
- **LIB-002 [P1] 素材情報を後から編集する** — 名前、タグ、お気に入り、評価、メモ、作成・更新日時、使用履歴、由来、不足項目を保持する。
- **LIB-003 [P0] Inboxを未整理素材の保全領域として使う** — 録音、取込み、分離結果などを失わず、試聴、名称変更、タグ、昇格、保管、削除、重複検出を行える。
- **LIB-004 [P1] 素材同士の関係を辿る** — 録音とラック、ラックと音色、音声と分析、素材と使用先などを相互に確認できる。

### 6.2 Projectと履歴

- **PRJ-001 [P0] 制作内容を完全に保存する** — 時間軸、トラック、ラック、プラグイン状態、MIDI、音声参照、支援履歴、表示、入出力、書出し設定を復元する。
- **PRJ-002 [P0] 自動保存を世代管理する** — 現行状態、未確定変更、復旧用世代を分け、破損時は最新の有効世代へ戻れる。
- **PRJ-003 [P1] 変更を取り消し、やり直す** — パラメーター、ラック、クリップ、トラック、素材情報、支援提案、取込み、削除を対象に履歴を操作できる。
- **PRJ-004 [P0] ファイルやプラグインが不足しても開く** — 不足一覧、再関連付け、置換、無視、無効な置き場所を使い、残りの制作内容を利用できる。
- **PRJ-005 [P1] 持ち運べる制作一式を作る** — 参照音声、MIDI、制作状態、ラック、代替音声、使用プラグイン、版、メモを収集し、プラグイン本体や権利情報を無断で複製しない。
- **PRJ-006 [P1] 保存形式を安全に更新する** — 形式の版を確認し、変換前に控えを作り、古い形式や新しい形式を推測で上書きしない。

### 6.3 AIによる制作支援

- **FLOW-006 [P1] 提案を確認してから適用する** — 対象、現在値、提案値、理由、効果、危険性を確認し、選択した変更だけを適用または破棄できる。
- **AI-001 [P0] 説明だけでは制作内容を変更しない** — 状態説明を求めても、セッション、ファイル、プロジェクトを変更しない。
- **AI-002 [P0] 提案は変更案として提示する** — 自動適用せず、対象、現在値、提案値、理由、効果、確信度、危険性を表示する。
- **AI-003 [P0] 確認済みの提案だけを適用する** — 選択適用、一括適用、破棄、試聴、取消しを提供し、不可逆な処理を自動実行しない。
- **AI-004 [P0] 参照する制作情報を選べる** — ラック、パラメーター、分析、クリップ、プロジェクト、メモ、音色、試聴音声、診断情報から対象を選択する。
- **AI-005 [P0] 外部送信の内容と目的を示す** — 送信元、送信先、目的、保持、音声の短縮・変換・匿名化を事前に示し、音声は明示操作でのみ送信する。

## 7. 安全性、復旧、製品品質

### 7.1 音声とデータの安全

- **G-003 [P0] 起動と復旧直後は安全にミュートする** — 起動、機器変更、復旧直後は出力を止め、安全な初期音量から明示操作で音を出す。
- **G-004 [P0] どの制作領域からも緊急ミュートできる** — 緊急操作で即座に無音となり、録音済みデータとセッションは変化しない。
- **G-005 [P0] 失敗の影響範囲を限定する** — プラグイン、機器、後処理、素材管理の一部が失敗しても、問題箇所、守られたデータ、復旧操作を示し、残りの制作内容を利用できる。
- **G-006 [P1] 空、処理中、失敗を区別する** — 状態を文言と形状で区別し、失敗時は原因、影響、次の操作を表示する。
- **AUD-004 [P0] 音声機器の切断を安全に処理する** — 再生または録音中に機器が失われると、音声処理を異常・ミュート状態へ移し、取得済みデータを読める状態で残す。
- **AUD-006 [P0] 異常な音声値とフィードバックを防ぐ** — 非数、無限値、過大音量、フィードバックの疑いを検出し、制限またはミュートして原因を表示する。
- **REC-004 [P0] 録音異常時に取得済みデータを保全する** — 画面停止、機器切断、記憶装置の遅延、プラグイン異常が起きても、欠落範囲と復旧状態を記録し、途中までのWAVを失わない。
- **DONE-002 [P0] 失敗を成功として残さない** — 録音データ消失、無音の成功表示、壊れた保存情報の既定値化、孤立した子プロセス、見えない外部送信を許さない。

### 7.2 復旧と隔離

- **FLOW-007 [P0] 障害から制作へ戻る** — プラグイン、機器、子プロセスの異常後に、再試行、迂回、セーフモード、復旧を選び、録音済みデータを保ったまま制作を続けられる。
- **REL-001 [P0] プラグイン障害を隔離・復旧する** — 問題のプラグインを特定し、終了、無効化、再試行、置き場所化を行って、残りのプロジェクトを利用できる。
- **REL-002 [P0] 音声子プロセスを孤立させない** — 画面の異常終了、再起動、子プロセス異常時に、安全に終了または親プロセス喪失を検出し、録音のロックを残さない。
- **REL-003 [P0] 後処理を中断・再開する** — 分析、書出し、分離、走査を画面と音声から分離し、中断した途中結果を完成扱いせず、再実行できる。
- **REL-004 [P0] 記憶装置の障害を扱う** — 容量不足、書込み失敗、破損、途中ファイルを明示し、取得済みデータと有効な保存世代を残す。

### 7.3 情報保護とWindows品質

- **SEC-001 [P0] 手元で完結する中核機能を守る** — 音声、プロジェクト、制作情報を明示許可なしに外部送信せず、オフラインでも演奏、録音、加工、配置、保存を利用できる。
- **SEC-003 [P1] 診断情報を安全に保存する** — 問題の特定に必要な情報だけを残し、音声、個人情報、秘密値、絶対パスを無制限に記録しない。
- **WIN-001 [P1] 表示倍率と複数画面で操作できる** — 表示倍率、画面移動、再接続、縦横比が変わっても主要操作が隠れず、プラグイン画面を扱える。
- **WIN-002 [P1] 設定を明示的に保存する** — 音声、MIDI、表示、素材、AI、情報保護、ショートカット、保存先を再起動後に復元する。

### 7.4 性能と完成判定

- **Q-001 [P0] 起動時間を測定できる** — 初回・再起動、機器の有無、走査中などの条件とWindows構成を添えて、操作可能になるまでの時間を記録する。
- **Q-002 [P0] 重い処理より音声を優先する** — 走査、支援、分析、素材検索、書出し中も、音声処理、緊急ミュート、録音を破綻させない。
- **Q-003 [P0] 長時間の制作を維持する** — 長時間の再生、録音、プラグイン、素材操作で、メモリー、ハンドル、CPU、音切れを許容範囲に保つ。
- **DONE-001 [P0] 三領域の制作循環を完成させる** — Playで音を作り、Designで素材化し、Arrangeで組み合わせ、保存・再起動・書出しまでを実機と回帰テストで確認し、残る制限を明示する。

## 8. 観測点

設計とテストでは、各要件について少なくとも一つの観測点を定める。

| 観測点 | 確認する内容                                                 |
| ------ | ------------------------------------------------------------ |
| 音     | 出力、無音、音量、同期、遅延、鳴り終わり                     |
| 画面   | 状態、選択、操作可能性、進捗、エラーの意味                   |
| データ | 原本、派生素材、保存情報、プロジェクト、素材索引、由来       |
| 履歴   | 取消し、やり直し、音色候補、提案、復旧世代                   |
| 境界   | プラグイン、音声機器、MIDI、記憶装置、外部サービスの失敗範囲 |

一つの要件が複数の技術層へまたがっても要求IDは分割せず、操作から結果までを同じ要件として追跡する。
